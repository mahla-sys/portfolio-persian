# 🎯 هدف پروژه

طراحی و توسعه یک سامانه مدیریت کتابخانه با دو سطح دسترسی:

- **کاربر:** مشاهده کتاب‌ها، امانت گرفتن کتاب، مدیریت اطلاعات شخصی
- **مدیر:** مدیریت کاربران، کتاب‌ها، نویسندگان، دسته‌بندی‌ها و فرآیند امانت

---

# 🏗 معماری نرم‌افزار

پروژه بر اساس **معماری سه‌لایه (Three-Layer Architecture)** توسعه داده شده است.

لایه‌های اصلی سیستم:

- Presentation
- Business Logic
- Data Access

این ساختار باعث جداسازی منطق برنامه، رابط کاربری و دسترسی به پایگاه داده شده و نگهداری و توسعه پروژه را ساده‌تر می‌کند.

# زیرساخت ارتباطی (Radio)

برای ارتباط بین بخش‌های مختلف سیستم، یک لایه پیام‌رسان داخلی (Radio) طراحی شده است.

در این معماری هر بخش تنها وظیفه خود را انجام می‌دهد و از جزئیات سایر بخش‌ها اطلاعی ندارد. ارتباط بین ماژول‌ها تنها از طریق پیام‌های رویدادمحور (Event-Driven) انجام می‌شود.

# معماری نرم‌افزار

سیستم بر اساس معماری **Event-Driven** توسعه داده شده است.

ماژول‌های اصلی عبارت‌اند از:

- مدیریت فیلترهای ورود
- مدیریت اجرای معاملات
- مدیریت موقعیت‌های باز
- مدیریت بستن معاملات
- مدیریت ریسک
- ثبت و تحلیل داده‌ها (Logger)

تمام ارتباطات بین این بخش‌ها از طریق سیستم Radio انجام می‌شود و وابستگی مستقیمی بین ماژول‌ها وجود ندارد.

📘 DailyFlow | سامانه مدیریت و تحلیل فعالیت‌های روزانه

وضعیت پروژه: در حال توسعه (Phase 1)

DailyFlow یک نرم‌افزار مدیریت وظایف و تحلیل فعالیت‌های روزانه است که با هدف تبدیل برنامه‌ریزی سنتی به یک سیستم دیجیتال، ماژولار و توسعه‌پذیر طراحی شده است.

✨ ایده اصلی پروژه

هدف این پروژه تنها ساخت یک To-Do List نیست.

در این سیستم، وظایف، احساسات، برنامه روزانه و داده‌های رفتاری کاربر به صورت ساختاریافته ثبت می‌شوند تا در آینده امکان تحلیل الگوهای رفتاری، اتوماسیون و استفاده از هوش مصنوعی فراهم شود.

معماری نرم‌افزار

پروژه بر اساس معماری Feature-Sliced Design (FSD) طراحی شده است تا با افزایش حجم پروژه نیز ساختار کد منظم و قابل توسعه باقی بماند.

لایه‌های اصلی سیستم:

App
Pages
Widgets
Features
Entities
Shared

وابستگی بین این لایه‌ها تنها در یک جهت انجام می‌شود تا از ایجاد وابستگی‌های پیچیده جلوگیری شود.
